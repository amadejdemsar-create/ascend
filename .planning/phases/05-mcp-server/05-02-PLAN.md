**1. Extend `goalService.list()` in `lib/services/goal-service.ts`:**

Add optional `skip` and `take` parameters to the `list` method signature:

```typescript
async list(userId: string, filters?: GoalFilters, pagination?: { skip?: number; take?: number }) {
  return prisma.goal.findMany({
    where: { /* existing filters */ },
    orderBy: [{ sortOrder: "asc" }, { createdAt: "desc" }],
    include: { category: true },
    ...(pagination?.skip !== undefined && { skip: pagination.skip }),
    ...(pagination?.take !== undefined && { take: pagination.take }),
  });
}
```

This is a backwards-compatible change; existing callers that do not pass pagination are unaffected.

**2. Complete `handleGoalTool` in `lib/mcp/tools/goal-tools.ts`:**

Implement the full switch statement for all 6 goal tools. Each case follows the same pattern:
1. Parse args with Zod v4 schema for runtime validation
2. Call the appropriate goalService method
3. Return `{ content: [{ type: "text", text: JSON.stringify(result, null, 2) }] }`

**create_goal:**
- Parse args with `createGoalSchema.parse(args)`
- Call `goalService.create(userId, parsed)`
- Return the created goal

**get_goal:**
- Extract `id` from args, validate it is a non-empty string
- Call `goalService.getById(userId, id)`
- If null, return `{ content: [{ type: "text", text: "Goal not found" }], isError: true }`
- Otherwise return the full goal with children, parent, category

**update_goal:**
- Extract `id` from args, validate it is a non-empty string
- Parse remaining args with `updateGoalSchema.parse()` (remove `id` from the parse input)
- Call `goalService.update(userId, id, parsed)`
- Return the updated goal

**delete_goal:**
- Extract `id` and `cascade` from args
- If `cascade` is true: fetch the goal with children via `goalService.getById(userId, id)`, then delete each child recursively (depth-first) before deleting the parent. Use a recursive helper function.
- If `cascade` is false or not provided: call `goalService.delete(userId, id)` which orphans children (sets parentId to null)
- Return confirmation message: `"Goal deleted successfully. ${cascade ? 'Children also deleted.' : 'Children orphaned (parentId set to null).'}"  `

**list_goals:**
- Parse filter args with `goalFiltersSchema.parse()` for the filter portion
- Extract `limit` (default 50, cap at 100) and `offset` (default 0) from args
- Call `goalService.list(userId, filters, { skip: offset, take: limit })`
- Return the list with a summary header: `"Found ${results.length} goals (offset: ${offset}, limit: ${limit})\n\n${JSON.stringify(results, null, 2)}"`

**search_goals:**
- Extract `query` from args, validate it is a non-empty string
- Call `goalService.search(userId, query)`
- Return results with count: `"Found ${results.length} goals matching '${query}'\n\n${JSON.stringify(results, null, 2)}"`

**Error handling for all cases:**
Wrap each case in try/catch. If a Zod validation error occurs, return `{ content: [{ type: "text", text: "Validation error: ${error.message}" }], isError: true }`. For other errors, return `{ content: [{ type: "text", text: "Error: ${error.message}" }], isError: true }`.
