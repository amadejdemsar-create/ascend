1. Add level formula functions to `lib/constants.ts`:
   - `xpForLevel(level: number): number` returning `100 * level * level` (Level 1 = 100 XP, Level 5 = 2500 XP)
   - `levelFromXp(totalXp: number): number` returning `Math.floor(Math.sqrt(totalXp / 100))`
   - `xpToNextLevel(totalXp: number): { current: number; needed: number; percentage: number }` computing progress within current level

2. Create `lib/services/gamification-service.ts` as a plain TS object module (matching existing service pattern):
   - `awardXp(userId: string, goalId: string, horizon: string, priority: string)`: Calculate XP as `Math.round(XP_PER_HORIZON[horizon] * PRIORITY_MULTIPLIER[priority])`. Create XpEvent record. Upsert UserStats (increment totalXp, increment goalsCompleted). Compute new level via `levelFromXp`. If newLevel > stats.level, update level. Also check and reset weeklyScore if weekStartDate is before current week's Monday (use `startOfWeek` from date-fns with `{ weekStartsOn: 1 }`). Increment weeklyScore by the XP amount. Return `{ amount, totalXp, level, leveledUp, weeklyScore }`.
   - `getStats(userId: string)`: Return UserStats with computed xpToNextLevel data. Return safe defaults if no UserStats exists.
   - `checkAndResetWeeklyScore(stats, now)`: Internal helper that resets weeklyScore to 0 and updates weekStartDate when the stored weekStartDate is before the start of the current week.
   - IMPORTANT: The `awardXp` function must NOT fetch the goal to check previous status. The caller (API route / MCP handler) is responsible for verifying the status transition. This keeps the gamification service focused on XP logic only.

3. Extend `prisma/schema.prisma` with recurring goal support:
   - Add `RecurringFrequency` enum: `DAILY`, `WEEKLY`, `MONTHLY`

   - Add index on `[recurringSourceId]`
