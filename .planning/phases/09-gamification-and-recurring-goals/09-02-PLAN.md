<objective>
Build the recurring goals system: a service that generates instances from recurring templates, tracks streaks on completion, detects missed deadlines, and provides UI for creating recurring goals.

Purpose: Recurring goals are the habit building core of Ascend. Users create a template (e.g., "Exercise" with WEEKLY frequency), and the system lazily generates the next instance whenever the current one is completed or when the dashboard checks for due instances. Consecutive completions build streaks, which feed into the gamification system.
Output: recurring-service.ts, two API routes, updated goal form with recurring fields.
</objective>

  <action>
1. Create `lib/services/recurring-service.ts` as a plain TS object module:
   - Import date-fns functions: `addDays`, `addWeeks`, `addMonths`, `startOfDay`, `differenceInCalendarDays`, `startOfWeek` (already installed v4.x)
   - `getNextDueDate(lastDate: Date, frequency: string, interval: number): Date`: Switch on frequency to call addDays/addWeeks/addMonths
   - `isStreakBroken(lastCompleted: Date, frequency: string, interval: number): boolean`: Compute nextDue from getNextDueDate, compare against today using startOfDay normalization. For DAILY: streak broken if today > nextDue + 1 day grace. For WEEKLY/MONTHLY: streak broken if today > nextDue.
   - `generateDueInstances(userId: string)`: Query all recurring template goals for the user (`isRecurring: true, recurringSourceId: null`). For each template: (a) check if an incomplete instance exists (`recurringSourceId: template.id, status: { in: ["NOT_STARTED", "IN_PROGRESS"] }`); if yes, skip. (b) Check if streak is broken since lastCompletedInstance; if so, reset currentStreak to 0 on the template. (c) Create a new Goal instance copying title (append date range label like "Week of Mar 31"), horizon, priority, categoryId, parentId from template. Set `recurringSourceId` to template.id. Set deadline to the next due date. Return array of created instances.
   - `completeRecurringInstance(userId: string, instanceId: string)`: Find the instance and its template (via recurringSourceId). Increment template's currentStreak by 1. Update longestStreak if currentStreak > longestStreak. Set lastCompletedInstance to now. Return updated template with streak data.
   - `listTemplates(userId: string)`: Return all recurring templates for the user with their current/longest streaks and last instance info.

2. Create `app/api/goals/recurring/route.ts`:
   - GET handler: authenticate, call `recurringService.listTemplates(userId)`, return JSON array
   - Follow the thin route handler pattern: auth + service call + response

3. Create `app/api/goals/recurring/generate/route.ts`:
   - POST handler: authenticate, call `recurringService.generateDueInstances(userId)`, return JSON with count and created instances
   - This endpoint is called by the dashboard on load to ensure due instances exist
