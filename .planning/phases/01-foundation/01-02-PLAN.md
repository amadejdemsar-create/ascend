  <action>
    1. **Write the complete prisma/schema.prisma** with all models from the research. The schema must include:

       **Generator block** (Prisma 7 syntax):
       ```prisma
       generator client {
         provider = "prisma-client"
         output   = "../generated/prisma"
       }
       datasource db {
         provider = "postgresql"
       }
       ```

       **Enums:**
       - `Horizon`: YEARLY, QUARTERLY, MONTHLY, WEEKLY
       - `GoalStatus`: NOT_STARTED, IN_PROGRESS, COMPLETED, ABANDONED
       - `Priority`: LOW, MEDIUM, HIGH

       **User model:**
       - id (cuid), email (optional, unique), name (optional), apiKey (unique, default cuid)
       - Relations: goals[], categories[], stats (optional UserStats)
       - Timestamps: createdAt, updatedAt

       **Goal model:**
       - id (cuid), userId (FK to User), title, description (optional)
       - horizon (Horizon enum), status (GoalStatus, default NOT_STARTED), priority (Priority, default MEDIUM)
       - Hierarchy: parentId (optional FK to Goal, self-reference "GoalHierarchy", onDelete SetNull), parent relation, children relation
       - Category: categoryId (optional FK to Category, onDelete SetNull), category relation
       - SMART fields: specific, measurable, attainable, relevant, timely (all optional strings)
       - Progress: progress (Int, default 0), targetValue (Float?), currentValue (Float?), unit (String?)
       - Dates: startDate, deadline, completedAt (all optional DateTime)
       - Ordering: sortOrder (Int, default 0)
       - Notes: notes (optional string)
       - Relations: progressLogs[], user (cascade delete)
       - Timestamps: createdAt, updatedAt
       - Indexes: userId, parentId, categoryId, [horizon, userId], [status, userId], deadline

       **Category model:**
       - id (cuid), userId (FK to User), name, color (default "#4F46E5"), icon (optional)
       - Hierarchy: parentId (optional FK to Category, self-reference "CategoryHierarchy", onDelete Cascade), parent relation, children relation
       - Ordering: sortOrder (Int, default 0)
       - Relations: goals[], user (cascade delete)
       - Timestamps: createdAt, updatedAt
       - Unique constraint: [userId, name, parentId]
       - Indexes: userId, parentId

       **ProgressLog model:**
       - id (cuid), goalId (FK to Goal, onDelete Cascade), value (Float), note (optional)
       - Timestamps: createdAt
       - Indexes: goalId, createdAt

       **UserStats model:**
       - id (cuid), userId (unique FK to User)
       - totalXp (Int, default 0), level (Int, default 1)
       - currentStreak (Int, default 0), longestStreak (Int, default 0)
       - lastActiveDate (optional DateTime), weeklyScore (Int, default 0)
       - weekStartDate (optional DateTime), goalsCompleted (Int, default 0)
       - Relations: user (cascade delete)
       - Timestamps: updatedAt

       **XpEvent model:**
       - id (cuid), userId, amount (Int), source (String), goalId (optional)
       - Timestamps: createdAt
       - Indexes: userId, createdAt

    2. **Create lib/db.ts** with the Prisma 7 singleton pattern using adapter-pg:
       ```typescript
       import { PrismaPg } from "@prisma/adapter-pg";
       import { PrismaClient } from "../generated/prisma/client";

       const adapter = new PrismaPg({
         connectionString: process.env.DATABASE_URL!,
       });

       const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

       export const prisma =
         globalForPrisma.prisma ||
         new PrismaClient({ adapter });

       if (process.env.NODE_ENV !== "production") {
         globalForPrisma.prisma = prisma;
       }
       ```

       IMPORTANT: The import path is `../generated/prisma/client`, NOT `@prisma/client`. This is a Prisma 7 requirement.

    3. **Generate Prisma client and run initial migration:**
       ```bash
       npx prisma generate
       npx prisma migrate dev --name init
       ```

       If running against a remote database, ensure DATABASE_URL in .env.local points to the Dokploy PostgreSQL internal URL (or a local PostgreSQL for development). If no local PostgreSQL is available, the migration will target the production database directly (acceptable for initial setup of a personal project).

    1. **Create prisma/seed.ts** that seeds the test user (idempotent with upsert):
       ```typescript
       import { PrismaPg } from "@prisma/adapter-pg";
       import { PrismaClient } from "../generated/prisma/client";
       import "dotenv/config";

       const adapter = new PrismaPg({
         connectionString: process.env.DATABASE_URL!,
       });
       const prisma = new PrismaClient({ adapter });

       async function main() {
         // Upsert test user (idempotent)
         const user = await prisma.user.upsert({
           where: { email: "amadej@ascend.local" },
           update: {},
           create: {
             email: "amadej@ascend.local",
             name: "Amadej",
             apiKey: process.env.API_KEY || "ascend-dev-key-change-me",
           },
         });

         console.log("Seeded user:", user.id, "apiKey:", user.apiKey);

         // Upsert initial stats
         await prisma.userStats.upsert({
           where: { userId: user.id },
           update: {},
           create: { userId: user.id },
         });

         console.log("Seeded user stats for:", user.id);
       }

       main()
         .catch((e) => {
           console.error(e);
           process.exit(1);
         })
         .finally(async () => {
           await prisma.$disconnect();
         });
       ```

       Note: The seed script creates its OWN PrismaClient instance (not using the lib/db.ts singleton) because the seed runs as a standalone script via `tsx`, not within the Next.js runtime.

    2. **Run the seed:**
       ```bash
       npx prisma db seed
       ```
       This uses the seed command configured in prisma.config.ts (`tsx prisma/seed.ts`).

    3. **Verify database integrity** by running a quick check:
       ```bash
       npx tsx -e "
         import { PrismaPg } from '@prisma/adapter-pg';
         import { PrismaClient } from './generated/prisma/client';
         import 'dotenv/config';
         const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL! });
         const prisma = new PrismaClient({ adapter });
         const user = await prisma.user.findFirst();
         console.log('User:', user?.name, user?.apiKey);
         const stats = await prisma.userStats.findFirst();
         console.log('Stats:', stats ? 'exists' : 'MISSING');
         await prisma.\$disconnect();
       "
       ```
