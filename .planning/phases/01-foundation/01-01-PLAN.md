       ```bash

       ```
       Then copy all generated files (except .git) from /tmp/ascend-scaffold/ into /Users/Shared/Domain/Code/Personal/goals/, preserving the existing .planning/ directory. Delete the temp scaffold after copying.

    2. **Update next.config.ts** to add `output: "standalone"` (required for Docker deployment):
       ```typescript
       import type { NextConfig } from "next";
       const nextConfig: NextConfig = {
         output: "standalone",
       };
       export default nextConfig;
       ```

    3. **Install Phase 1 dependencies:**
       ```bash
       npm install @prisma/client @prisma/adapter-pg pg dotenv zod
       npm install -D prisma tsx @types/pg
       ```

    4. **Initialize Prisma:**
       ```bash
       npx prisma init --datasource-provider postgresql
       ```
       This creates prisma/schema.prisma and .env. After init, update the schema's generator block:
       ```prisma
       generator client {
         provider = "prisma-client"
         output   = "../generated/prisma"
       }
       datasource db {
         provider = "postgresql"
       }
       ```
       Note: In Prisma 7, the datasource block does NOT contain a `url` field. The URL is in prisma.config.ts.

    5. **Create prisma.config.ts** at the project root:
       ```typescript
       import "dotenv/config";
       import { defineConfig, env } from "prisma/config";
       export default defineConfig({
         schema: "prisma/schema.prisma",
         migrations: {
           path: "prisma/migrations",
           seed: "tsx prisma/seed.ts",
         },
         datasource: {
           url: env("DATABASE_URL"),
         },
       });
       ```

    6. **Create the placeholder page** at app/page.tsx: A simple page that displays "Ascend" with a subtitle "Goal tracking, deployed." and the current timestamp. Use only Tailwind CSS for styling. Dark background (#0F0F14), centered text, Inter font.

    7. **Create health check endpoint** at app/api/health/route.ts:
       ```typescript
       import { NextResponse } from "next/server";
       export async function GET() {
         return NextResponse.json({ status: "ok", timestamp: new Date().toISOString() });
       }
       ```

    8. **Create the Dockerfile** following the pattern from the research. Multi-stage build: deps, builder (with prisma generate), runner. Copy Prisma artifacts (generated/, prisma/, prisma.config.ts, relevant node_modules). Use CMD that runs prisma migrate deploy before starting the server:

       ```
       CMD ["sh", "-c", "npx prisma migrate deploy && node server.js"]
       ```

    9. **Update .gitignore** to include: generated/, .env, .env.local, .env.production

    IMPORTANT: Use Prisma 7 patterns throughout. The provider is "prisma-client" (NOT "prisma-client-js"). The import path will be "../generated/prisma/client" (NOT "@prisma/client").

    **Step 2: Create Application service** in Dokploy:
