  <action>
    1. **Create lib/services/hierarchy-helpers.ts** implementing the validateHierarchy function:
       - Takes userId, parentId, and childHorizon as parameters
       - Looks up the VALID_PARENT_HORIZONS map from constants.ts
       - If the child horizon expects null parent (YEARLY), throws "YEARLY goals cannot have a parent"
       - Fetches the parent goal from the database, verifies it belongs to the user
       - Checks the parent's horizon matches the expected parent horizon
       - Throws descriptive error if mismatch: "A WEEKLY goal must have a MONTHLY parent, but the specified parent is YEARLY"

    2. **Create lib/services/goal-service.ts** as a plain object with async methods (not a class). Every method takes `userId: string` as the first parameter:

       - `list(userId, filters?)`: Find many goals with optional filters for horizon, status, categoryId, parentId. Order by sortOrder asc then createdAt desc. Include category relation.
       - `create(userId, data)`: Validate hierarchy if parentId provided (call validateHierarchy). Create goal with userId.
       - `getById(userId, id)`: Find first goal matching id AND userId. Include category, children (ordered by sortOrder), and parent.
       - `update(userId, id, data)`: If changing parentId or horizon, re-validate hierarchy. Verify ownership. Update.
       - `delete(userId, id)`: Verify ownership. Delete (children get parentId set to null via onDelete: SetNull).
       - `getTree(userId)`: Fetch all yearly goals with nested includes 3 levels deep (yearly > quarterly > monthly > weekly). Include category at each level.
       - `search(userId, query)`: Use Prisma `contains` on title and description (case-insensitive with `mode: "insensitive"`). Filter by userId.
       - `logProgress(userId, goalId, data)`: Verify goal ownership. Create ProgressLog entry. Update goal's currentValue (increment by data.value). If targetValue exists, recalculate progress percentage.
       - `getProgressHistory(userId, goalId)`: Verify goal ownership. Return all ProgressLog entries ordered by createdAt desc.

    3. **Create lib/services/category-service.ts** as a plain object with async methods:

       - `list(userId)`: Find all categories for user, ordered by sortOrder. Include children.
       - `listTree(userId)`: Fetch all categories, build tree in memory (top-level = parentId null, nest children recursively). Return as nested array.
       - `create(userId, data)`: Create category with userId.
       - `getById(userId, id)`: Find first category matching id AND userId. Include goals count.
       - `update(userId, id, data)`: Verify ownership. Update.
       - `delete(userId, id)`: Verify ownership. Delete (cascades to children via onDelete: Cascade, goals get categoryId set to null).

    4. **Create lib/tree-queries.ts** with raw SQL recursive CTE utilities:
       - `getDescendants(goalId, userId)`: Recursive CTE that walks down from a goal to all its children/grandchildren. Returns array of {id, title, parentId, horizon, progress, depth}. Filters by userId.
       - `getAncestors(goalId, userId)`: Recursive CTE that walks up from a goal to its parent chain. Returns ancestors ordered by depth (root first).

       Use `prisma.$queryRaw` with tagged template literals for parameterized queries. Use the exact SQL patterns from the research. Cast enum columns with `::text` to get string values from PostgreSQL.

    IMPORTANT: Import prisma from "@/lib/db" (the singleton). Import types from the generated client path. All service methods are async and return Prisma model types.
    IMPORTANT: Do NOT add try/catch blocks in service functions. Let errors propagate to the API route handlers, which will format them as HTTP responses.
