**In lib/mcp/schemas.ts**, append 10 todo tool definitions to the TOOL_DEFINITIONS array, after the existing Context tool definitions. Add a `// ── Todos ──` section comment. Use the same JSON Schema style as existing tools. The tool definitions:

1. **create_todo**: properties: title (string, required), description (string), priority (string, enum LOW/MEDIUM/HIGH), goalId (string, "Parent goal ID to link this to-do to"), categoryId (string), dueDate (string, format date-time, ISO 8601), scheduledDate (string, format date-time), isRecurring (boolean), recurrenceRule (string, "iCalendar RRULE string e.g. FREQ=DAILY or FREQ=WEEKLY;BYDAY=TU,TH"). Required: ["title"].

2. **get_todo**: properties: id (string, required). Returns full to-do with linked goal.

3. **update_todo**: properties: id (string, required), title (string), description (string), priority (string, enum), status (string, enum PENDING/DONE/SKIPPED), goalId (string), categoryId (string), dueDate (string, format date-time), scheduledDate (string, format date-time), sortOrder (integer), isBig3 (boolean), big3Date (string, format date-time). Required: ["id"].

4. **delete_todo**: properties: id (string, required).

5. **list_todos**: properties: status (string, enum PENDING/DONE/SKIPPED), priority (string, enum), categoryId (string), goalId (string), dateFrom (string, format date-time), dateTo (string, format date-time), isBig3 (boolean, "Filter for Big 3 to-dos only"), limit (integer, default 50), offset (integer, default 0). No required fields.

6. **complete_todo**: properties: id (string, required). Description: "Mark a to-do as completed. Triggers side effects: awards XP based on priority, updates recurring streak if applicable, and auto-increments linked goal progress."

7. **search_todos**: properties: query (string, required, "Search text to match against titles and descriptions").

8. **get_daily_big3**: properties: date (string, format date-time, "Date to get Big 3 for (defaults to today, ISO 8601)"). No required fields.

9. **set_daily_big3**: properties: todoIds (array of strings, required, "Up to 3 to-do IDs to mark as today's priorities"), date (string, format date-time, "Date to set Big 3 for (defaults to today, ISO 8601)"). Required: ["todoIds"].

10. **get_todos_for_date**: properties: date (string, required, format date-time, "Date to get to-dos for (ISO 8601)"). Required: ["date"].

**Create lib/mcp/tools/todo-tools.ts** following the exact same pattern as context-tools.ts:
- Import todoService from "@/lib/services/todo-service"
- Import createTodoSchema, updateTodoSchema, todoFiltersSchema from "@/lib/validations"
- Import ZodError from "zod"
- Define McpContent type (same as context-tools.ts)
- Export async function handleTodoTool(userId, name, args) returning Promise<McpContent>
- Wrap all cases in try/catch; ZodError returns validation error, other errors return generic error message
- 10 switch cases:

**create_todo**: Parse args through createTodoSchema. If args.isRecurring or args.recurrenceRule, add those to the create data (createTodoSchema does not include recurrence fields, so extract isRecurring and recurrenceRule from args before parsing, and spread them into the data object passed to todoService.create after Zod parsing). Actually, looking at the service, create() only accepts CreateTodoInput fields. For recurrence, the executor should set isRecurring and recurrenceRule by first creating the to-do and then calling todoService.update() with those fields since updateTodoSchema does not include them either. **Simpler approach:** Since the Prisma create accepts any field on the model, and todoService.create only passes CreateTodoInput fields through, handle recurrence by doing a follow-up update after create if isRecurring/recurrenceRule are present. Use todoService.create(userId, zodParsed) for the base fields, then if args.isRecurring or args.recurrenceRule, call todoService.update(userId, created.id, { isRecurring: Boolean(args.isRecurring), recurrenceRule: String(args.recurrenceRule) }) to set recurrence. Return the final state via todoService.getById(userId, created.id).

**get_todo**: Validate id is non-empty string. Call todoService.getById(userId, id). Return 404-style error if null.

**update_todo**: Extract id from args, parse remaining with updateTodoSchema. Call todoService.update(userId, id, parsed).

**delete_todo**: Validate id. Call todoService.delete(userId, id). Return confirmation message.

**list_todos**: Map args to filter format: if args.isBig3 is boolean true/false, convert to string "true"/"false" for todoFiltersSchema. Parse through todoFiltersSchema. Extract limit/offset for pagination (default 50/0). Call todoService.list(userId, filters, { skip: offset, take: limit }).

**complete_todo**: Validate id. Call todoService.complete(userId, id). Return the result including _xp and _streak metadata.

**search_todos**: Validate query is non-empty string. Call todoService.search(userId, query).

**get_daily_big3**: If args.date provided, parse as Date. Call todoService.getBig3(userId, date).

**set_daily_big3**: Validate todoIds is array of strings. If args.date provided, parse as Date. Call todoService.setBig3(userId, todoIds, date).

**get_todos_for_date**: Validate date string. Call todoService.getByDate(userId, new Date(date)).

All return JSON.stringify(result, null, 2) wrapped in McpContent format.
