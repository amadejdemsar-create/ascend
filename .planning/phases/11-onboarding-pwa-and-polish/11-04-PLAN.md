  <action>
    1. Update `next.config.ts`:
       - Add `experimental: { viewTransition: true }` to the config object
       - Keep existing `output`, `serverExternalPackages`, and `headers()` (added by Plan 02)

    2. Update `app/(app)/layout.tsx`:
       - Import `ViewTransition` from 'react'
       - Wrap `{children}` inside the main tag with `<ViewTransition>{children}</ViewTransition>`
       - This gives automatic cross-fade transitions on route changes within the app shell
       - The ViewTransition component is a no-op in browsers that don't support the View Transitions API

    3. Add view transition CSS to `app/globals.css`:
       - Add `::view-transition-old(.page-transition)` with fade-out animation (200ms ease-out)
       - Add `::view-transition-new(.page-transition)` with fade-in animation (200ms ease-in)
       - Add keyframes for fade-out (opacity 1 to 0) and fade-in (opacity 0 to 1)
       - Add a subtle slide-up animation for entering content:
         `::view-transition-new(*)` with translateY(4px) to translateY(0) over 200ms
       - Add hover micro-interaction classes:
         `.hover-lift` class that adds `transition: transform 150ms ease, box-shadow 150ms ease` and on hover applies `transform: translateY(-2px)` and increased shadow
         `.hover-glow` class for interactive elements with a subtle scale(1.02) and glow effect on hover
       - Add smooth progress bar transition: `.progress-bar-animated` with `transition: width 600ms cubic-bezier(0.4, 0, 0.2, 1)`

    4. Create `lib/hooks/use-animated-counter.ts`:
       - "use client" module
       - Export `useAnimatedCounter(target: number, duration?: number): number`
       - Default duration: 600ms
       - Uses `useState` and `useEffect` with `requestAnimationFrame`
       - Ease-out cubic: `1 - Math.pow(1 - progress, 3)`
       - On target change, animate from current value to new target
       - If target is 0, set immediately (no animation from 0 to 0)
       - Check `window.matchMedia('(prefers-reduced-motion: reduce)')` at the start. If reduced motion preferred, set value instantly without animation.
       - Return the current animated integer value
  </action>
