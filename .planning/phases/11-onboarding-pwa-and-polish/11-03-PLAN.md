       - Define `OutboxEntry` interface: { id: string, url: string, method: string, headers: Record<string, string>, body: string, timestamp: number }
       - DB_NAME = 'ascend-outbox', STORE_NAME = 'mutations', DB_VERSION = 1
       - `getDb()`: Opens/creates the database with the mutations object store (keyPath: 'id')
       - `enqueue(entry: Omit<OutboxEntry, 'timestamp'>)`: Adds a mutation to the queue with current timestamp
       - `drain()`: Gets all entries sorted by timestamp (oldest first), replays each sequentially. On successful replay, deletes the entry. On failure (fetch throws), breaks the loop (still offline). Returns the number of successfully drained entries.
       - `getPendingCount()`: Returns the count of entries in the outbox (for UI indicator if needed)
       - Use `db.put(STORE_NAME, ...)` for enqueue, `db.getAll(STORE_NAME)` for drain, `db.count(STORE_NAME)` for pending count
       - Sort entries by timestamp before draining to ensure correct ordering

       - In the fetch handler:
         a) For GET requests matching CACHED_API_ROUTES (check `url.pathname.startsWith(route)`):
            - Try fetch first (network-first)
            - On success: clone response, open API_CACHE, put the clone keyed by request URL, return original response
            - On network failure (catch): try caches.match(request), return cached response if found, otherwise return a Response with 503 status and JSON body { error: "Offline", offline: true }
         b) For non-GET requests (POST, PATCH, DELETE): pass through without caching (client-side outbox handles failures)
         c) For all other requests: pass through normally
       - In the activate handler: also delete the API_CACHE if it exists alongside old version caches
