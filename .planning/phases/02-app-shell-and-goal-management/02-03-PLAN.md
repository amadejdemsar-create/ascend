API endpoints (from Phase 1):
- GET /api/goals?horizon=X&status=X&categoryId=X&parentId=X - list goals
- POST /api/goals - create goal (body: CreateGoalInput)
- GET /api/goals/[id] - get goal with children
- PATCH /api/goals/[id] - update goal (body: UpdateGoalInput)
- DELETE /api/goals/[id] - delete goal
- GET /api/categories - list categories as tree

    **1. Create lib/hooks/use-goals.ts** ("use client"):
    - Import useQuery, useMutation, useQueryClient from @tanstack/react-query.
    - Import queryKeys from lib/queries/keys.
    - Import types: GoalFilters, CreateGoalInput, UpdateGoalInput from lib/validations.
    - Get API key from `process.env.NEXT_PUBLIC_API_KEY!` and create a shared `headers` object with Content-Type and Authorization Bearer.
    - Export `useGoals(filters?: GoalFilters)`: query that fetches `/api/goals` with filter params as URLSearchParams. Query key: `queryKeys.goals.list(filters)`.
    - Export `useGoal(id: string)`: query that fetches `/api/goals/${id}`. Query key: `queryKeys.goals.detail(id)`. Enable only when id is truthy.
    - Export `useCreateGoal()`: mutation that POSTs to `/api/goals`. On success, invalidate `queryKeys.goals.all()`.
    - Export `useUpdateGoal()`: mutation that PATCHes `/api/goals/${id}`. Accepts `{ id: string; data: UpdateGoalInput }`. On success, invalidate both `queryKeys.goals.all()` and `queryKeys.goals.detail(id)`.
    - Export `useDeleteGoal()`: mutation that DELETEs `/api/goals/${id}`. On success, invalidate `queryKeys.goals.all()`.
    - All fetch calls should check `!res.ok` and throw an Error with the response error message.
    - Follow the exact patterns from the research doc's Pattern 4.

    **2. Create lib/hooks/use-categories.ts** ("use client"):
    - Export `useCategories()`: query that fetches `/api/categories` (returns tree structure). Query key: `queryKeys.categories.tree()`.
    - Use same auth headers pattern.
  </action>
