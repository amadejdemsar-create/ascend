**2. Create lib/services/todo-recurring-service.ts:**

Follow the structural pattern of recurring-service.ts but use rrule library.

Import: `import { prisma } from "@/lib/db"`, `import { rrulestr } from "rrule"`, `import { startOfDay, subDays } from "date-fns"`

Export `todoRecurringService` object with:

**`getOccurrences(rruleString, start, end)`**: Parse rrule, return `rule.between(start, end, true)`. Used by calendar.

**`generateDueInstances(userId)`**: Find templates (isRecurring=true, recurringSourceId=null, status=PENDING). For each: parse rrule, get next occurrence via `rule.after(startOfDay(new Date()), true)`, check if pending instance exists for that date, if not create one. Before creating, check if streak is broken: get the previous occurrence via `rule.before(startOfDay(new Date()), false)`, if that date exists and is after lastCompletedDate, reset currentStreak to 0. Return created instances.

**`completeRecurringInstance(userId, instanceId)`**: Find instance and template. Increment streak (currentStreak + 1, longestStreak = max). Calculate consistency score: count completed instances in last 30 days (`status: "DONE", completedAt >= subDays(now, 30), recurringSourceId = template.id`). Count expected occurrences via `rule.between(subDays(now, 30), now).length`. Score = round((completed/expected) * 100), clamped 0 to 100. Handle edge case where expected=0 (score=100). Update template with new streak data and consistency score. Return streak data object.

**`getStreakData(userId, templateId)`**: Simple fetch of template's streak fields.

**3. Add Big 3 methods to todo-service.ts:**

Add import: `import { todoRecurringService } from "@/lib/services/todo-recurring-service"`

Add these methods to the todoService object:

**`getBig3(userId, date?)`**: date defaults to today. Query `prisma.todo.findMany({ where: { userId, isBig3: true, big3Date: startOfDay(date) }, orderBy: { sortOrder: "asc" }, include: { category: true, goal: { select: { id: true, title: true } } } })`.

**`setBig3(userId, todoIds: string[], date?)`**: date defaults to today.
1. Validate: if todoIds.length > 3, throw "Maximum 3 Daily Big 3 allowed"
2. Validate: verify all todoIds belong to the user via `prisma.todo.findMany({ where: { id: { in: todoIds }, userId } })`. If any missing, throw "One or more to-dos not found".
3. Unset existing Big 3 for that date: `prisma.todo.updateMany({ where: { userId, isBig3: true, big3Date: startOfDay(date) }, data: { isBig3: false, big3Date: null } })`
4. Set new Big 3: `prisma.todo.updateMany({ where: { id: { in: todoIds } }, data: { isBig3: true, big3Date: startOfDay(date) } })`
5. Return the updated Big 3 to-dos

**`getByDate(userId, date)`**: Get all to-dos for a specific date (for calendar day view). Query where `OR: [{ scheduledDate: startOfDay(date) }, { dueDate: startOfDay(date), scheduledDate: null }]`, status not filtered (show all). Include category and goal. Order by isBig3 desc (Big 3 first), then sortOrder asc.

**`getByDateRange(userId, start, end)`**: Get all to-dos in a date range (for calendar month). Query where `OR: [{ scheduledDate: { gte: start, lte: end } }, { dueDate: { gte: start, lte: end }, scheduledDate: null }]`. Include category and goal. Return the flat list (caller groups by date).

**`bulkComplete(userId, ids: string[])`**: Loop through ids calling `this.complete(userId, id)`. Collect results. Return array of completed to-dos. Wrap in a try/catch per item so one failure does not block others.

**`reorder(userId, items: { id: string; sortOrder: number }[])`**: Same pattern as goalService.reorderGoals: `prisma.$transaction(items.map(...))`.

**app/api/todos/recurring/generate/route.ts** (POST):
