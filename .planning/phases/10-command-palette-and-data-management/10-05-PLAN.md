  <name>Task 1: Configure Dokploy PostgreSQL backup schedule</name>
  <action>
Configure automated pg_dump backups in the Dokploy dashboard for the Ascend PostgreSQL database.

**Steps for the user (or via Dokploy MCP tools if available):**

1. Open the Dokploy personal dashboard
2. Navigate to the Ascend project's PostgreSQL service
3. Go to the Backups tab
4. Configure a backup schedule:
   - Frequency: Daily at 03:00 UTC (cron expression: `0 3 * * *`)
   - Retention: Keep last 7 backups (1 week of daily snapshots)
   - Destination: Local storage on the VPS (S3 offload optional for future)
5. Run a manual backup to verify the configuration works
6. Confirm the backup file was created and is non-trivially sized (indicates actual data was dumped)

**Why this is a checkpoint:**
Dokploy backup configuration requires interacting with the Dokploy dashboard UI or using Dokploy MCP tools with the correct database service ID. The database service ID must be retrieved from the existing Dokploy project configuration, which varies per deployment.

**Alternative via MCP:**
If the executor has access to `mcp__dokploy-personal__*` tools, they can:
1. List projects to find the Ascend project
2. Find the PostgreSQL service ID
3. Configure backup via the appropriate Dokploy API endpoint (if backup endpoints are exposed via MCP)
If backup configuration is not available via MCP, manual dashboard configuration is required.
