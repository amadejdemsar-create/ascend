1. Create `lib/hooks/use-keyboard-shortcuts.ts`:
   - "use client" at top
   - Import `useUIStore` from `@/lib/stores/ui-store`
   - Import `useTheme` from `next-themes`
   - Import `useRouter` from `next/navigation`
   - Accept a single argument: `onToggleShortcutRef: () => void` (callback to open the shortcut reference)
   - useEffect with document.addEventListener("keydown"):
     a. **Guard**: If `e.target` is an HTMLElement with tagName INPUT, TEXTAREA, SELECT, or isContentEditable is true, return immediately (do not process shortcuts)
     b. **Guard**: If metaKey or ctrlKey is pressed (modifier combos), return (let Cmd+K and browser shortcuts pass through unmodified). Exception: Cmd+K is handled by the command palette, not here.
     c. Switch on `e.key`:
        - "1": `e.preventDefault()`, `setActiveView("cards")`, `router.push("/goals")`
        - "2": `e.preventDefault()`, `setActiveView("list")`, `router.push("/goals")`
        - "3": `e.preventDefault()`, `setActiveView("board")`, `router.push("/goals")`
        - "4": `e.preventDefault()`, `setActiveView("tree")`, `router.push("/goals")`
        - "5": `e.preventDefault()`, `setActiveView("timeline")`, `router.push("/goals")`
        - "n": `e.preventDefault()`, `openGoalModal("create")`
        - "b": `e.preventDefault()`, `toggleSidebar()`
        - "t": `e.preventDefault()`, cycle theme (light -> dark -> system -> light, same cycle as ThemeToggle)
        - "?": `e.preventDefault()`, call `onToggleShortcutRef()`
        - "g" then "d": This is complex to implement with chords. SKIP chord shortcuts. Just use "d" for dashboard: `e.preventDefault()`, `router.push("/")`
        - "s": `e.preventDefault()`, `router.push("/settings")`
   - Cleanup: remove event listener on unmount
   - Include all store methods and router in the dependency array

2. Create `components/command-palette/keyboard-shortcuts.tsx`:
   - "use client" directive
   - Import Dialog, DialogContent, DialogHeader, DialogTitle from `@/components/ui/dialog`
   - Accept props: `open: boolean`, `onOpenChange: (open: boolean) => void`
   - Define shortcut data as a static array of groups:
     a. **Navigation**: `1` Cards, `2` List, `3` Board, `4` Tree, `5` Timeline, `d` Dashboard, `s` Settings
     b. **Actions**: `n` New Goal, `b` Toggle Sidebar, `t` Toggle Theme
     c. **Global**: `Cmd+K` / `Ctrl+K` Command Palette, `Esc` Close Dialogs, `?` This Reference
   - Render a Dialog with the shortcut groups as a styled grid
   - Each shortcut displays the key in a `<kbd>` element (monospace, border, rounded, slight background) and the description next to it
   - Use a two column grid within each group: key on left, description on right
   - Style the kbd elements with: `inline-flex items-center justify-center rounded border border-border bg-muted px-1.5 py-0.5 font-mono text-xs font-medium text-muted-foreground`
