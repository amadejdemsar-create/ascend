   - `cd /Users/Shared/Domain/Code/Personal/ascend && npm install --save-dev @types/pdfkit`

2. Update `next.config.ts`:
   - Add `"pdfkit"` to the `serverExternalPackages` array alongside `"@modelcontextprotocol/sdk"`
   - This ensures PDFKit's .afm font metric files are included in the standalone build

3. Create `lib/services/export-service.ts`:
   - Import `goalService` and `categoryService`
   - Import PDFDocument from "pdfkit"
   - Import Document, Paragraph, TextRun, HeadingLevel, Packer from "docx"

   a. `exportJSON(userId: string): Promise<string>`:
      - Fetch goals and categories via Promise.all
      - Return `JSON.stringify({ exportedAt, goals, categories }, null, 2)`

   b. `exportCSV(userId: string): Promise<string>`:
      - Extract the csvEscape and formatCSV logic from data-tools.ts (or import if already extracted)
      - Fetch goals, cast to Record<string, unknown>[], return formatCSV result
      - CSV headers: id, title, horizon, status, priority, progress, targetValue, currentValue, unit, deadline, categoryId, parentId, createdAt

   c. `exportMarkdown(userId: string): Promise<string>`:
      - Extract formatMarkdown logic from data-tools.ts (or import if already extracted)
      - Fetch goals, return formatMarkdown result

   d. `exportPDF(userId: string): Promise<Buffer>`:
      - Fetch goals and categories via Promise.all
      - Create new PDFDocument({ margin: 50 })
      - Collect chunks via doc.on("data"), resolve Buffer.concat on doc.on("end")
      - Title: "Ascend Goal Report" in 24pt Helvetica-Bold
      - Subtitle: "Exported: {date}" in 10pt Helvetica
      - moveDown(2)
      - For each horizon (YEARLY, QUARTERLY, MONTHLY, WEEKLY):
        - Section heading in 16pt Helvetica-Bold
        - For each goal in that horizon:
          - Checkbox text: "[x]" or "[ ]" based on COMPLETED status
          - Title, priority in parentheses, progress percentage
          - Progress bar: draw a filled rectangle proportional to progress (e.g., 200px wide bar, fill color #4F46E5 for progress, #E5E7EB for remainder, 10px tall)
        - moveDown(1) between horizons
      - Add a summary section at the bottom: total goals, completed count, completion rate
      - doc.end()
      - Return the collected buffer

   e. `exportDOCX(userId: string): Promise<Buffer>`:
      - Fetch goals and categories
      - Build Document with sections:
        - Title paragraph: "Ascend Goal Report" as TITLE heading
        - Date paragraph: "Exported: {date}"
        - For each horizon: HEADING_2 paragraph, then for each goal: paragraph with checkbox character (Unicode ballot box U+2610 / checked U+2611), bold title, priority and progress
      - Call `Packer.toBuffer(doc)` and return the result (it returns a Buffer)

4. Update `lib/mcp/tools/data-tools.ts`:
   - Replace local `csvEscape`, `formatCSV`, and `formatMarkdown` with imports from `@/lib/services/export-service` (export them from export-service.ts)
   - Alternatively, move these utility functions to a shared `lib/services/export-helpers.ts` file that both data-tools.ts and export-service.ts import from. Choose whichever approach is cleaner. The goal is zero duplication of CSV/Markdown formatting logic.

5. Create `app/api/export/route.ts`:
   - GET handler: validate API key
   - Read `format` query param (default "json")
   - Switch on format:
     - "json": call exportJSON, return with Content-Type "application/json", Content-Disposition "attachment; filename=ascend-export.json"
     - "csv": call exportCSV, return with Content-Type "text/csv", Content-Disposition "attachment; filename=ascend-export.csv"
     - "markdown": call exportMarkdown, return with Content-Type "text/markdown", Content-Disposition "attachment; filename=ascend-export.md"
     - "pdf": call exportPDF, return with Content-Type "application/pdf", Content-Disposition "attachment; filename=ascend-report.pdf"
     - "docx": call exportDOCX, return with Content-Type "application/vnd.openxmlformats-officedocument.wordprocessingml.document", Content-Disposition "attachment; filename=ascend-report.docx"
     - default: return 400 with error "Invalid format. Supported: json, csv, markdown, pdf, docx"
   - Return as `new NextResponse(data, { headers: { "Content-Type": ..., "Content-Disposition": ... } })`
