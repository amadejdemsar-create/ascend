# Wave 8 Close-Out Security Audit — `ascend-security`

**Date:** 13. 5. 2026
**Scope:** Cumulative security audit on Wave 8 (Phases 1 through 9).
**Verdict:** PASS WITH NOTES. Zero wave-close blockers. Two medium notes for Wave 8b hardening.

## Threat model coverage

| Threat | Verdict | Evidence |
|--------|---------|----------|
| T1. Cross-tenant data leak (workspaceId scoping) | PASS | All 18 workspace-scoped models declare `workspaceId String` (NOT NULL). Every modified service includes `{ userId, workspaceId }` in its where clauses. 5 intentionally user-only models (`UserStats`, `XpEvent`, `FocusSession`, `Session`, `UserSettings`) documented inline. |
| T2. CRDT auth bypass | PASS | Token route authenticates first, then `assertCanEditBlockDocument` enforces WRITE_NODE + ownership. Persist route accepts only `x-crdt-secret` via timing-safe compare. Hocuspocus `onAuthenticate` enforces `documentName === "blockdoc:" + entryId`. `persistFromCrdt` bypass documented at DZ-23/DZ-24 with the persist-secret boundary. |
| T3. Activity feed cross-tenant leak | PASS | `activityEventService.list` checks `WorkspaceMembership ACTIVE` before reading. Route enforces `workspaceId === auth.workspaceId`. |
| T4. MCP workspaceId override | PASS | Zero occurrences of `workspaceId` in tool input schemas. Zero occurrences of `args.workspaceId` in handlers. `workspaceId` flows only from the factory binding. |
| T5. Secrets handling | PASS WITH NOTES | `CRDT_JWT_SECRET` ≥32 chars enforced at module init. Distinct env vars. Timing-safe compare for persist secret. No leaked values in error responses (`CRDT service temporarily unavailable` returned instead). Note: `CRDT_PERSIST_SECRET` lacks an explicit ≥32 char minimum. |
| T6. Token refresh path | PASS | `provider.setConfiguration({ token }) + provider.sendToken()`. Old token expires naturally (5 min TTL). Proactive refresh scheduled at expiresAt-30s. `authenticationFailed` triggers the fallback path. |
| T7. Presence / awareness data exposure | PASS | Awareness broadcasts `username` (= name OR email OR "Anonymous") and a deterministic color only. No password hash, no API key, no internal IDs beyond userId. |
| T8. Workspace settings route | PASS | PATCH requires MANAGE_WORKSPACE permission + ownership. Zod-validated. DELETE returns 403 unconditionally. |
| T9. Workspace switcher membership lookup | PASS | `listMembers` user select is explicit (`id, email, name`); no sensitive fields surface. |
| T10. MCP tools workspaceId binding | PASS | Same as T4. `get_workspace` rejects mismatched id with isError. `get_activity_events` passes only the factory-bound workspaceId to the service. |

## Medium notes for Wave 8b hardening

1. **`CRDT_PERSIST_SECRET` lacks minimum length enforcement** in `apps/crdt/src/persist.ts:20` and `lib/auth.ts:verifyCrdtPersistSecret`. Add `|| CRDT_PERSIST_SECRET.length < 32` guards before the CRDT server is internet-facing in Wave 8b.

2. **MCP CORS allows all origins**. `Access-Control-Allow-Origin: *` is fine for header-based auth but should be tightened to an allowlist in Wave 8b when multi-user expands the attack surface.

## Informational notes

- `_resolveWorkspaceId` trusts the JWT claim without re-checking membership. Standard JWT behavior; mitigated by `permissionService` on mutations. In Wave 8b, add session revocation on membership removal so the token loses access on next refresh.
- No enforcement that `AUTH_JWT_SECRET`, `CRDT_JWT_SECRET`, `CRDT_PERSIST_SECRET` are distinct at startup. Add a module-level guard in Wave 8b.

## Pre-launch checks for Wave 8b multi-user

1. Integration test with two users (A and B in different workspaces): every data endpoint as B targeting A's data should return 403/404.
2. CRDT cross-document test: issue token for entry X, attempt to connect to `blockdoc:Y`. Expect Hocuspocus rejection.
3. Membership revocation test: remove a membership, verify next JWT refresh returns 403 via `permissionService`.
4. Secret distinctness check at startup.
5. CORS tightening on `/api/mcp`.
6. Rate limiting on `/api/crdt/token` (currently unlimited).
