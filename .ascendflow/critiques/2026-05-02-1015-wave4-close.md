## Must-Fix Items (PRD-scoped, currently NOT DONE)

### 1. Missing Upload button in /context toolbar
- **Checks**: PQ2, PQ11
- **File**: `apps/web/app/(app)/context/page.tsx:310-326` (PageHeader actions only contain View Switcher + "New" button)
- **PRD**: C2 explicitly calls for "Upload button in /context toolbar opens a file picker (multiple selection allowed)"
- **Impact**: Drag-and-drop is the only upload affordance today. A user who doesn't know the feature exists will never discover it. Notion has explicit "Upload or embed" in the slash menu *and* a sidebar entry; Slack has "+"; every file-capable product has a visible button.
- **Fix**: Add an `Upload` icon button next to "New" in the PageHeader actions. On click, open `<input type="file" multiple>`, pipe selected files through `useUploadFile({ createEntry: true })`. ~15 minutes.

### 2. No spreadsheet table preview
- **Checks**: PQ13, PQ1
- **File**: `apps/web/components/editor/file-block.tsx:157` (spreadsheet MIME falls to generic FileCard)
- **PRD**: C5 explicitly calls for "Spreadsheet: small table preview (first 5 rows × 5 cols) + Open full table link"
- **Impact**: CSV/XLSX uploads currently show only generic file card. The data IS available (extractedText is "row 1: a, b, c\n...") but the UI doesn't render it as a table. Notion renders CSVs as inline databases; Drive shows preview grids.
- **Fix**: Add `SpreadsheetPreview` component that parses first 5 lines of `extractedText` into a 5×5 `<table>`. Route MIME `text/csv` and `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` to it in `file-block.tsx`. ~30 minutes.

## Should-Fix (Next Sprint)

- **Timestamp-linked transcript seeking** (audio/video). Otter.ai/Loom parity. Already noted in BACKLOG as deferred (no Whisper word-level timestamps schema field).
- **Image alt-text editing**. ImageBlock displays alt but no inline edit affordance. Notion has caption fields.
- **Upload progress indicator**. Drop → overlay disappears → toast on completion. No intermediate signal for large files. `useUploadFile` doesn't expose upload bytes. Slack/Notion show progress.
- **Drag-drop discoverability**. No visual hint, tooltip, or empty-state copy that the page accepts file drops. Add a "Tip: drag files here" line in the empty state.
- **Lightbox inconsistency**. `FileBlockImage` (inside file-block.tsx) has a simple open/close overlay; `ImageBlock` (image-block.tsx) has full arrow-key nav, zoom, counter. Files inserted as FileNode get a degraded lightbox vs ImageNode. Unify.
- **Image presigned URL expiry**. R2 URLs expire in 5 minutes; `image-block.tsx` already comments this risk. Long reading sessions will see broken images. Refresh on visibility change, or proxy through an always-fresh API route.

## Bright Spots

- **Drop zone**: drag counter pattern for nested elements is correctly implemented; deferring to editor's FileDropPlugin via `e.defaultPrevented` is a detail many miss.
- **MIME dispatch in file-block.tsx**: clean delegation to specialized sub-components, no shared state leakage.
- **ImageBlock lightbox** (image-block.tsx): arrow-key nav, Fit/100%/200% zoom, document-order collection, counter, Escape close. Apple Photos quality.
- **Error copy**: "File too large for extraction (X MiB exceeds Y MiB limit)" not raw error objects.
- **Auto-poll status** (useFileStatus with refetchInterval that stops on terminal): Linear-quality background job feedback.
- **Security**: all URLs through `/api/files/[id]`, sandboxed PDF iframe, SSRF allowlist on MCP upload, dual-layer MIME + size validation.
- **Slash menu**: logical grouping (formatting → structure → files → AI), distinct fuzzy-match keywords, MIME-filtered file pickers.

## Summary

Wave 4 delivers a functional, well-architected file system that covers the core user story at a solid quality level. The implementation quality of what IS shipped is genuinely good: error handling is human, keyboard nav works, accessibility attributes are present, architecture is clean, security is strong.

**However**: two PRD-scoped success criteria (C2: Upload button, C5: Spreadsheet preview) are NOT DONE. Per the Execution Quality Bar (no silent spec simplification), these must be addressed or explicitly deferred with user acknowledgment before the wave can close. Both are <1h of work combined.

**Verdict**: GOOD passes the wave-close gate, but the two NOT DONE PRD criteria require resolution.
