## Must-fixes (resolved in `ea2ba04`)

1. **Internal nomenclature in user-facing tooltips.** Three strings referenced "Wave 8" / "Wave 8b" in tooltips users would see. Replaced with human-readable equivalents:
   - "Sole workspace cannot be deleted in Wave 8." → "You need at least one workspace. Create another before deleting this one."
   - "Coming in Wave 8b" → "Invitations are not yet available"
   - "Multi-user actor filter coming in Wave 8b" → "Filtering by actor will be available when collaboration launches"

## Should-fixes (BACKLOG)

- **Activity feed entity links don't deep-link.** Clicking "Project Roadmap" lands on `/context` (domain page), not the specific entry. Requires URL-param-based deep linking infrastructure across goal / todo / context list views. Tracked as Wave 8b priority.
- **`LINK_CREATED` / `LINK_REMOVED` rows render "entry to entry"** instead of the actual titles. Renderer should consume `fromTitle` / `toTitle` from the payload.
- **Connection indicator label "Offline (autosave)"** is slightly misleading when the user is online but Hocuspocus is unreachable. Consider "Local save" or "Saving locally".
- **Mobile activity feed has no filter affordance.** Filter sidebar is `hidden md:block`; add a sheet / drawer trigger for mobile.

## Wave 8b priorities

1. **Deep-linking infrastructure.** Every entity page needs URL-param navigation. Activity feed, browser history, link sharing, and mobile deep links all depend on this.
2. **Invite flow and multi-member UX.** Fill in the disabled buttons. This is the headline Wave 8b feature.
3. **Activity feed enrichment.** Add NODE_UPDATED events (most common user action), entity-title in link events, and deep links from every row.
4. **Jump-to-cursor for presence.** Click a presence avatar → scroll editor to that user's cursor. Essential for real collaboration.

## Bright spots

- The Hocuspocus integration is production-grade: separate app, separate secrets, separate Docker image, valid TLS, WebSocket lifecycle, proactive token refresh, graceful degradation to legacy autosave.
- `useRealtimeDocument` lifecycle management is thorough (token → provider → y-indexeddb → awareness → cleanup).
- Presence avatars + collaborative cursors are polished with reduced-motion respect, deterministic colors, WCAG-compliant contrast, and explicit `aria-label`s.
- Activity feed empty state is one of the best in the app: icon, message, actionable description.
- `ConnectionStatusIndicator` uses `role="status"` for screen-reader announcements.

## Final verdict

**GOOD.** Once the three tooltip copy fixes landed (`ea2ba04`), Wave 8 meets the close-out bar. The deep-linking gap and mobile-filter affordance are post-close polish, tracked in BACKLOG.md for Wave 8b.
