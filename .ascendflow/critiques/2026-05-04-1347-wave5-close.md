## Must-Fix Items (addressed pre-close)

### 1. Type-change kebab item was a stub
- **Initial state**: `table-header-cell.tsx` "Change type" item showed `toast.info("Type change UI will be expanded in a future phase")`.
- **PRD criterion**: "No data loss on schema changes — changing a field type prompts the user to confirm a coercion."
- **Fix shipped pre-close**: New `change-type-popover.tsx` dialog with allowed-targets list (TEXT → URL/EMAIL/PHONE, NUMBER → TEXT, SELECT → MULTI_SELECT) and force-convert flow. New POST route `/api/databases/[id]/fields/[fieldId]/change-type`. New hook `useChangeFieldType`.

### 2. No delight on milestones
- **Initial state**: Database creation, row creation, view switch were silent — no animation, no confetti, no positive reinforcement.
- **VISION quality bar**: "World-class, even Elon Musk would want to use this."
- **Fix shipped pre-close**:
  - View switch fade-slide animation in `database-detail.tsx` via `motion-safe:animate-in motion-safe:fade-in-0 motion-safe:slide-in-from-bottom-1` (respects `prefers-reduced-motion`).
  - Database-creation confetti via new `apps/web/lib/confetti.ts` (canvas-confetti, 60 particles, 2s cooldown, reduced-motion-aware). Fires from `useCreateDatabase.onSuccess`.

### 3. Relation autocomplete in Table cells was effectively broken
- **Initial state**: `RelationEditor` in Table cells received empty `resolvedEntries` and undefined `onSearch` — opening the popover and typing did nothing.
- **Fix shipped pre-close**: New `relation-cell-wrapper.tsx` that wires `useRelationSearch` (hits `/api/context/search`) and `useResolvedEntries` (resolves entry IDs to titles, 60s cache). Table view uses the wrapper for RELATION field cells.

## Bright Spots

- **Optimistic updates everywhere**: Board column drag, Calendar date drag, Timeline move + edge resize. Each pre-patches React Query cache, mutates, rolls back on error.
- **Error boundaries on every view** (DZ-7 compliance).
- **Empty states with Quick-pick fallbacks** in Board, Calendar, Timeline. One-click sets the required config without making the user open the view config popover.
- **Filter builder recursive AND/OR** with depth cap, visual combinator toggle ("All of" / "Any of"), per-field-type operator menu.
- **Formula error display** matches Notion: `#ERROR` cell with tooltip showing the expression and error message.
- **TanStack Virtual + sticky header + sticky primary column** in Table view.
- **Board drag overlay rotation** (`rotate-[2deg]`) gives tactile feel matching Linear's UX.
- **Week starts on Monday** in Calendar view (matches global locale rule).
- **Accessibility above average**: every interactive element has `aria-label`, table uses proper ARIA roles, Board cards handle Enter/Space, resize handles use `role="separator"`.
