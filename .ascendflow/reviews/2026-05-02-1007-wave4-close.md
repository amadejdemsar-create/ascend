## Non-Blocking Recommendations

1. **DZ-13 R2 orphan path** — `fileService.uploadBytes` writes to R2 first, then creates the Prisma row. If Prisma fails, the R2 object is orphaned permanently. `cleanupOrphanPending` only deletes PENDING-status rows. Recommend adding an R2 reconciliation script or extending cleanup to scan for R2 keys without matching File rows. Low risk in production (single server). Tracked in BACKLOG (MEDIUM).

2. **Migration idempotency wording** — `20260427000001/migration.sql:58` uses `ALTER TABLE "File" ADD CONSTRAINT "File_contextEntryId_fkey"` without a DO/IF NOT EXISTS guard. Safe because Prisma tracks migration state, but inconsistent with the "IDEMPOTENT" comment at the top. Not actionable now.

3. **Unstaged typecheck fix** — `packages/editor/package.json` + `pnpm-lock.yaml` are unstaged (added `@types/node` to packages/editor for the round-trip script). Must be committed with the wave-close.

4. **Duplicated `verifyCronSecret`** — Identical implementation in `extract/run/route.ts` and `cleanup/route.ts`. Tracked in BACKLOG (MEDIUM): extract to shared `lib/auth.ts` helper.
